项目一：天籁银发项目

  • 项目背景：
    面向退伍老兵及社区老人群体的听力障碍筛查，涵盖龙岗医院与社区行动。

  • 技术栈：
    Spring Cloud、Spring Boot、MyBatis、Redisson、Redis、MySQL、RocketMQ

  • 核心功能：

    • 听力筛查流程设计（状态机控制分贝调整）

    • 微信小程序第三方平台开发

    • 微信/支付宝支付集成及超时订单处理

    • 限额秒杀活动（针对退伍老兵助听器发放）

    • PC端扫码登录、验证码登录及防刷机制

    • 日志筛查统计并发送邮件

    • PC后台省市区志愿者机构树状结构展示

  • 技术难点与解决方案：

    • 秒杀超卖问题：使用 Lua 脚本和 Redis SETNX 优化，避免分布式锁并发瓶颈

    • 支付并发问题：状态机 + 乐观锁确保订单状态一致性

    • 筛查流程乱序：引入状态机流程控制

  • 创新亮点：

    • 状态机+乐观锁统一订单状态判定流程

    • 使用懒汉式缓存防止数据库击穿

    • Redis/Lua 脚本替代分布式锁解决秒杀并发性能问题

  • 量化指标：

    • 秒杀服务响应能力提升 300%

    • 订单状态误判率降至 0%

  • 个人贡献：

    • 项目整体架构设计、核心功能编码、关键并发问题解决全部由本人独立完成

项目二：亲友同屏（畅听助手 App 子系统）

  • 项目背景：
    中国联通 & 中国聋人协会联合开发，专为听障人士提供辅助沟通服务。

  • 技术栈：
    Spring Cloud、Spring Boot、MyBatis、Redisson、Redis、MySQL、RocketMQ、Socket.IO

  • 核心功能：

    • 实时视频共享 & 语音转文字系统

    • 聊天室功能（消息展示+辅助转译）

    • 风险内容识别与防诈骗机制

  • 技术难点与解决方案：

    • Socket.IO 无法跨集群通信：使用 Redis 发布订阅机制构建统一通信平台

    • 正则敏感词过滤低效：引入 Trie 前缀树进行敏感词检索加速

  • 创新亮点：

    • Redis Pub/Sub 实现 Socket.IO 跨集群通信

    • 敏感词过滤系统由正则迁移为 Trie 树结构，响应时间明显优化

  • 量化指标：

    • RT 降低 50%，Socket 连接数提升 200%

    • 敏感词过滤效率提升 30%

  • 个人贡献：

    • 完成分布式通信架构改造

    • 敏感词过滤系统重构

项目三：莉美易 SaaS 平台

  • 项目背景：
    面向美容行业的多租户客户管理系统

  • 技术栈：
    Spring Cloud、Spring Boot、MyBatis、Redisson、Redis、MySQL、RocketMQ、Socket.IO

  • 核心功能：

    • 店务管理：员工排班打卡、客服聊天、提成设置

    • 订单管理：商品/卡项/退款流程、支付对接

    • 用户管理：客户等级、预约等

    • 促销活动：三人成团拼团系统

    • 商品卡项：SKU/SPU、库存、会员卡/次卡管理

    • 权限控制：老板、店长、美容师等角色权限划分

    • 数据分析：T+1日报表、门店盈亏分析

  • 技术难点与解决方案：

    • 第三方打卡地图接口改为使用 Redis GEO 进行定位打卡

    • 每日数据统计耗时高，使用 CompletableFuture 异步编排

  • 创新亮点：

    • Redis GEO 实现低成本、高效打卡定位

    • 异步任务优化日报生成耗时

  • 量化指标：

    • 异步编排方案使日报统计耗时降低 80%

  • 个人贡献：

    • 独立设计与开发拼团促销系统

    • 权限系统、Redis GEO、数据统计逻辑优化与落地
