Java并发面试题

July 03, 2025 2067

🧵 一、线程基础与通信

1. Thread 和 Runnable 区别
Thread是类，Runnable是接口；Thread是重写run()，Runnable实现run()；推荐用Runnable实现，便于资源共享。

2. sleep、wait、join、yield 区别

  • sleep：暂停线程，不释放锁；

  • wait：等待并释放锁，需notify/notifyAll唤醒；

  • join：等待某线程执行完；

  • yield：让出CPU，但可能立刻被抢回。

3. 线程间通信方式
常用方式包括：共享变量、wait/notify、Condition、BlockingQueue、Thread.join()。

4. 如何查看死锁
使用 jstack 命令查看线程堆栈，找“Found one Java-level deadlock”。

5. 守护线程
后台线程，JVM无用户线程时自动退出。如GC线程。

6. 线程生命周期与状态
NEW -> RUNNABLE -> BLOCKED/WAITING/TIMED_WAITING -> TERMINATED。

二、并发基础概念

7. 并发、并行、串行区别

  • 并发：多个任务交替执行（单核也可）；

  • 并行：多个任务同时执行（多核）；

  • 串行：任务一个接一个执行。

8. 并发三大特性
原子性、可见性、有序性。

9. 对线程安全的理解
多线程访问共享资源时数据一致，行为符合预期；通过同步手段保障。

10. 如何保证线程安全
使用Synchronized、Lock、volatile、线程安全集合、避免共享等。

11. 如何保证线程执行顺序
可用join、CountDownLatch、CyclicBarrier、Condition、信号量Semaphore。

🔒 三、锁机制与同步

12. Synchronized 锁升级过程
偏向锁 → 轻量级锁 → 重量级锁，随着竞争加剧逐级升级。

13. Synchronized vs ReentrantLock

  • 前者为关键字，后者为类；

  • Lock更灵活（中断、公平锁、条件等待）；

  • Lock需手动释放，Synchronized自动释放。

14. ReentrantLock：lock() vs tryLock()

  • lock()：阻塞直到拿到锁；

  • tryLock()：尝试拿锁，不阻塞（可带超时时间）。

15. 公平锁 vs 非公平锁实现

  • 公平锁：线程按顺序排队（队列）

  • 非公平锁：新线程可插队；默认非公平，提高吞吐量。

16. CountDownLatch vs Semaphore 区别

  • CountDownLatch：一次性倒计时，常用于线程等待；

  • Semaphore：许可控制，可限流或资源控制，支持释放。

四、AQS 与原理

17. AQS 是什么
AbstractQueuedSynchronizer，用于构建锁和同步器的基础框架。通过CLH队列 + state状态变量 实现独占、共享锁。

18. AQS 实现可重入锁原理
ReentrantLock用AQS实现；state记录重入次数，当前线程重复加锁不会阻塞。

💡 五、ThreadLocal 专题

19. ThreadLocal 底层原理
每个线程维护一个ThreadLocalMap，key是ThreadLocal对象，value是变量副本。

20. ThreadLocal 内存泄露及避免
key为弱引用，value为强引用，未清理时容易泄露；**建议使用完调用remove()**。

☕ 六、volatile 与原子性

21. volatile vs synchronized 区别

  • volatile：只保证可见性、有序性，不保证原子性；

  • synchronized：三性全保证。

22. DCL为何需要volatile（1&2）
防止指令重排导致对象未初始化完成就被引用。

七、线程池原理

23. 为什么使用线程池
减少线程创建销毁开销、统一管理资源、提高系统响应速度。

24. 线程池参数解释
核心线程数、最大线程数、队列、拒绝策略、线程工厂。

25. 线程池执行流程
任务来时：核心线程未满→直接执行；满了→入队列；队列满→创建新线程；线程满→执行拒绝策略。

26. FixedThreadPool 用的阻塞队列？
使用 LinkedBlockingQueue，无界队列。

27. 提交任务时队列满了，会发生什么
执行拒绝策略（默认是AbortPolicy，抛异常）。

28. 线程池复用原理
线程执行完任务不销毁，等待新任务，从队列中取继续执行。

29. 队列为什么在线程前创建？
提高资源利用率，避免频繁创建销毁线程。

30. 阻塞队列作用
缓存任务，降低线程竞争；当线程繁忙时进行排队处理。

转为char数组，使用Arrays.sort(char[])；或自定义排序规则。
