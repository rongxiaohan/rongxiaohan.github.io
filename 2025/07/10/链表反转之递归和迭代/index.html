链表反转之递归和迭代

🌟 示例：反转单链表

给定一个单链表 1 -> 2 -> 3 -> 4 -> 5 -> null，反转后变为 5 -> 4 -> 3 -> 2 -> 1 -> null

定义节点结构如下（Java 举例）：

迭代法（Iterative）

思路：

  • 用三个指针：

    • prev：前一个节点

    • curr：当前节点

    • next：临时保存下一个节点

  • 每次把当前节点 curr.next 指向 prev

  • 最后返回 prev（新的头节点）

优点：

  • 时间复杂度：O(n)

  • 空间复杂度：O(1)，不需要额外空间

递归法（Recursive）

思路：

  • 递归地反转 head.next 后，让 head.next.next = head，然后 head.next = null

  • 每次递归返回的是反转后的新头结点

  2. 回溯过程中将 2.next.next = 2（3 → 2），再断开 2.next = null

  3. 将 1.next.next = 1（2 → 1），再断开 1.next = null

总结对比

方法 时间复杂度 空间复杂度 优点 缺点
迭代法 O(n) O(1) 效率高，实用性强 稍显“机械”
递归法 O(n) O(n) 简洁、易于理解 有栈溢出风险（深链）
