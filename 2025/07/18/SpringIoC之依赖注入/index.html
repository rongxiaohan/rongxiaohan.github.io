🌱 一、什么是依赖注入？

依赖注入（DI）是一种设计模式，指由容器负责创建对象，并将它所依赖的其它对象注入进去，而不是对象自己创建或查找依赖。

✨ 二、依赖注入的方式

注入方式 示例 是否支持循环依赖 特点
构造器注入 @Autowired Constructor ❌ 不支持 推荐方式（final+不可变性）
Setter注入 @Autowired setB(B b) ✅ 支持 可选依赖、可变性高
字段注入 @Autowired private B b ✅ 支持 最常用，但不利于测试
接口注入（罕见） 实现接口并由容器回调 ❌ 罕见 Spring中几乎不用

⚙️ 三、依赖注入的底层流程（以字段注入为例）

Spring 在执行 createBean() 时：

  1. 实例化 Bean（无参构造）

  2. 依赖注入（populateBean）

  3. 初始化（init-method / @PostConstruct）

  4. 放入一级缓存，完成注册

其中，populateBean() -> AutowireByType -> resolveDependency()：

  • 根据字段类型去容器中找 Bean

  • 找到就通过反射注入

  • 找不到抛出 NoSuchBeanDefinitionException

📦 四、常见注解与使用技巧

✅ 核心注解

注解 说明
@Autowired 自动按类型注入（默认 required = true）
@Qualifier("beanName") 与 @Autowired 联合使用，按名称限定
@Inject（JSR-330） Java 标准注入（功能类似）
@Resource(name="beanName") 按名称注入（来自 JSR-250）
@Value("${...}") 注入配置值或表达式

🚨 注意事项

  • 有多个同类型 Bean 时，需配合 @Qualifier 或使用 @Primary

  • 如果希望字段为可选，可以加 @Autowired(required=false)

  • 若使用构造器注入，Spring 会自动识别唯一构造器，无需 @Autowired

🧠 五、依赖注入 vs 控制反转

  • 控制反转（IoC） 是一种思想：控制权从程序员交给了框架。

  • 依赖注入（DI） 是实现 IoC 的一种方式。

除了 DI，IoC 还可以通过 Service Locator 模式实现，但 Spring 大部分用的是 DI。

🛠 六、源码核心类一览

类名/方法 作用说明
DefaultListableBeanFactory BeanFactory 实现，负责依赖查找
AutowiredAnnotationBeanPostProcessor 解析 @Autowired 注解
DependencyDescriptor 表示注入点的元信息（字段/参数等）
resolveDependency() 实际执行依赖查找和注入

📌 八、面试常问点总结

问题 回答要点
Spring中有哪些注入方式？ 构造器、字段、Setter、接口注入
Autowired 默认是按什么注入的？如何按名称？ 默认按类型，可用 @Qualifier 限定名称
多个同类型 Bean 怎么注入？ @Qualifier / @Primary / @Autowired(required=false)
Spring 如何处理循环依赖？ 三级缓存，仅支持单例+字段注入
Autowired 是怎么实现注入的？ 反射 + BeanPostProcessor 机制（AutowiredAnnotationBPP）
Autowired 和 Resource 区别？ Autowired 默认按类型，Resource 默认按名称
