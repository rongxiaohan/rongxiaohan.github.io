✅ 一、Nacos 集群部署目标

功能目标 描述
高可用 任一节点宕机不影响服务
可扩展 支持横向扩容
配置/注册双功能 同时作为配置中心 + 注册中心
支持持久化 使用 MySQL 存储配置和注册数据
易于管理 可通过控制台/运维工具管理节点

✅ 二、部署架构图

所有节点共用一个 MySQL，数据同步靠内部 Raft 协议

✅ 四、Nacos 集群管理方式

1. 管理 Web 控制台

  • 登录 Web 界面：http://x.x.x.x:8848/nacos

  • 默认用户名/密码：nacos/nacos

  • 可查看配置列表、服务注册信息、集群节点状态

2. 节点扩容缩容

✅ 扩容：

  • 修改所有节点的 cluster.conf 增加新 IP

  • 在新节点配置一致后，startup.sh -m cluster

✅ 缩容：

  • 停掉某个节点

  • 手动修改其他节点的 cluster.conf，删除对应条目

Nacos 会基于 Raft 协议自动完成 leader 选举、数据同步

3. 运维监控方式

方式 工具/描述
日志监控 nacos/logs 目录
健康检查 Nginx 反向代理失败回退
Prometheus 监控 开启 metrics 模块采集数据
容器管理 使用 K8s + Helm + StatefulSet

✅ 五、常见问题排查

问题 原因 解决方法
启动卡住 MySQL 无法连接 确保 IP、账号、端口、防火墙正确
注册失败 配置无效 检查注册方式是否配置为 cluster 模式
节点不互通 集群失败 确保节点间 8848 可访问，关闭防火墙
config 丢失 MySQL 无 binlog 开启 binlog + GTID
控制台慢 DNS 解析问题 /etc/hosts 配置本地 IP 映射

✅ 六、进阶建议

场景 建议
云原生部署 使用 K8s + Helm 安装 nacos-cluster
持久化性能优化 MySQL 增加主从，读写分离
高可用注册中心 搭配 Spring Cloud Gateway 做注册发现容错
灰度/多环境配置 使用 namespace + group 管理环境隔离配置
