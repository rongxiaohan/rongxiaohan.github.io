一、SkyWalking 是什么？

Apache SkyWalking 是一个 面向云原生的 APM（应用性能监控）与可观测性平台，支持分布式链路追踪、服务拓扑分析、慢请求定位等功能，专为微服务、容器、Service Mesh 等现代架构设计。

支持 Java、Go、PHP、Python、Node.js
支持 Spring Cloud、Dubbo、gRPC、K8s、Istio
可观测性三大支柱之一：Trace + Metrics + Topology

四、核心功能详解

1. 服务拓扑图

  • 显示服务之间调用关系图，谁调用了谁，谁耗时高

  • 可以快速找到瓶颈服务或超时依赖

2. Trace 链路追踪

  • 类似 Zipkin、Jaeger，按调用顺序记录链路详情

  • 可以查看一次请求从网关 → 用户服务 → 订单服务 → 库存服务 全流程的耗时、异常

3. Metrics 指标图

  • 每个服务的 QPS、响应时间、异常率等指标都有默认图表

  • 支持自定义仪表板

4. 错误聚合 & 分析

  • 可以聚合同类异常（如 NullPointer、Hystrix 超时）并高亮报错调用栈

5. 慢请求分析

  • 自动统计耗时最长的接口和调用链，精准定位瓶颈

五、常见应用场景

场景 SkyWalking 价值
微服务调用链路过长或失败 Trace 一键查看全链
某个接口很慢或频繁超时 慢调用分析 + 服务拓扑图
多服务交叉部署频繁发布 自动标记版本调用图
需要排查分布式事务异常 多服务 Trace 跟踪每个事务调用情况
开发人员 Debug 请求流程 Sky UI 中搜索 traceId 或日志关键词
自动监控系统健康度 设置 SLA 告警、RT 超标告警

七、实战技巧

实用技巧 方法
通过 traceId 定位问题 在日志中打印 traceId（从 MDC 里取）
对接 Nginx 或 Gateway SkyWalking 也能接入网关请求和跨服务 header
多环境隔离 不同环境用不同 service_name 或 OAP 端口
性能问题排查 排查调用链耗时最长的 Segment
自定义插件 可写 Agent 插件插桩业务方法（例如自定义 RPC 框架）

八、替代 vs 组合

场景 推荐工具
只需要链路追踪 SkyWalking（优于 Zipkin）
同时监控 JVM、内存、线程等指标 Prometheus + SkyWalking
精细日志分析 ELK + SkyWalking
K8s 监控、Istio SkyWalking 支持 Mesh Trace，也可配合 Grafana

总结

SkyWalking = 链路追踪 + 服务拓扑 + 应用性能分析 + Trace UI + 指标聚合平台

它在 Spring Cloud、Dubbo、K8s 等体系中是链路观测首选。
