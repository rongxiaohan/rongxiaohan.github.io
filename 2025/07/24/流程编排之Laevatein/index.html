流程编排之Laevatein

July 24, 2025 1878

一、Laevatein 是什么？

Laevatein 是一个由阿里开源的轻量级异步流程编排框架，主要用于解决如下问题：

  • 多步骤业务编排（如物流、CRM、订单处理等）

  • 多服务依赖、异步回调、状态流转

  • 适配微服务架构下的“流程复杂性”

它本质上是一个有状态的异步编排状态机引擎，具备如下特点：

特性 描述
异步流程编排 支持串行、并行、条件分支、重试、补偿等逻辑
状态持久化 支持将流程状态持久化到数据库、Redis 等
事件驱动 触发器基于事件机制，支持多种触发方式
轻量、无侵入 适配现有业务服务，不需要大改动

二、菜鸟或新手如何使用 Laevatein 进行流程编排？

以一个典型的物流流程为例（如：下单 → 分拣 → 打包 → 发货 → 配送 → 签收），Laevatein 可以对该流程建模并实现自动流转。

1. 定义流程结构（Flow DSL）

2. 实现业务 Action

3. 启动流程

4. 异常重试/补偿机制

Laevatein 支持配置失败自动重试、失败回滚补偿等机制。

三、使用场景举例（物流系统）

场景 编排价值
快递发货流程 自动化流转、多步骤异步调用，提高效率
客户退货流程 条件判断 + 异步处理，减少人为干预
异常包裹处理流程 分支流转 + 人工干预节点支持
仓内作业流程（拣货、打包） 可视化流程配置，灵活维护流程

四、适合“菜鸟”的入门建议

推荐起步方式：

  • 从 几个固定步骤 的流程开始建模（如订单流程）

  • 使用 YAML DSL 进行流程定义，清晰直观

  • 使用 内存 + 日志 模式跑通第一版，避免上来就集成 Redis/MySQL

必学模块：

  • FlowEngine

  • Action 接口及 execute() 实现

  • 状态持久化接口（可扩展）

  • 异步执行框架对接（如线程池、事件总线）
