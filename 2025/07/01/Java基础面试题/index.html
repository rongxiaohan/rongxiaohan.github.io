✅ 1. == 和 equals 的区别

  • == 比较的是引用地址是否相等

  • equals 比较的是内容是否相等（可重写）

  • 对于基本类型，== 比较的是值；引用类型默认按地址比，除非重写 equals

✅ 2. ConcurrentHashMap 的扩容机制

  • JDK8 使用分段 CAS + synchronized 控制每个桶

  • 扩容时采用协作转移（transfer）机制

  • 多线程抢占迁移数据，底层用 ForwardingNode 占位

✅ 3. ConcurrentHashMap 原理及 JDK7 vs JDK8 区别

特性 JDK7 JDK8
并发控制 Segment 分段锁 CAS + synchronized
底层结构 Segment → HashEntry[] Node[] + 链表/红黑树
扩容方式 全表加锁 多线程协作扩容
是否支持 null key ❌ 不支持

✅ 4. CopyOnWriteArrayList 原理

  • 采用 写时复制（Copy-On-Write）

  • 每次写操作都复制整个数组 → 写开销大

  • 读操作无锁、线程安全，适用于读多写少场景

✅ 5. final 关键字作用

  • 修饰变量：值不可变（引用不可变但对象内容可变）

  • 修饰方法：不可重写

  • 修饰类：不可被继承

  • 编译器可做优化，如内联替代等

✅ 6. hashCode 与 equals 的关系

  • 若两个对象 equals 相等，则它们的 hashCode 必须相等

  • 若两个对象 hashCode 相等，不一定 equals 相等

  • 用于 HashMap、HashSet 判断键值是否重复

✅ 7. HashMap 扩容机制原理

  • 扩容条件：元素数 > 容量 × 负载因子

  • 每次扩容容量翻倍

  • 重新 hash 和分桶（JDK8 中桶内链表长度 > 8 且容量 > 64 时转红黑树）

✅ 8. HashMap vs Hashtable

特性 HashMap Hashtable
线程安全 ❌ 否 ✅ 是（所有方法 synchronized）
性能 高 低（锁粒度粗）
是否允许 null ✅ 允许一个 null key 和多个 null value ❌ 都不允许
推荐 ✅ 现代项目首选，配合 ConcurrentHashMap

✅ 9. Java 异常体系

  • Throwable 是顶级父类，分为：

    • Error：严重错误，程序无法处理（如 OOM）

    • Exception：程序可以处理

      • Checked：必须处理（如 IOException）

      • Unchecked：运行时异常（如 NullPointerException）

✅ 10. JDK vs JRE vs JVM

名称 含义 功能
JDK Java Development Kit Java 开发环境（包含 JRE + 编译工具）
JRE Java Runtime Environment Java 运行环境（包含 JVM + 核心库）
JVM Java Virtual Machine Java 程序运行的虚拟机（平台无关）

✅ 11. List vs Set

特性 List Set
是否有序 ✅ 有 ❌ 一般无序（TreeSet 除外）
是否可重复 ✅ 是 ❌ 否
常用实现 ArrayList、LinkedList HashSet、TreeSet

✅ 12. String / StringBuffer / StringBuilder

类 是否可变 线程安全 用途
String ❌ 不可变 ✅ 安全 适合少量拼接、常量值
StringBuffer ✅ 可变 ✅ 安全 多线程场景下的拼接
StringBuilder ✅ 可变 ❌ 不安全 单线程高性能拼接 ✅

✅ 13. StringBuffer vs StringBuilder

  • 都是可变字符序列，功能类似

  • StringBuffer 方法带 synchronized，线程安全但慢

  • StringBuilder 不加锁，适合单线程

✅ 14. JDK1.7 vs 1.8 中 HashMap 的底层变化

特性 JDK1.7 JDK1.8
桶结构 链表 链表 + 红黑树
扩容方式 头插法（易死循环） 尾插法（顺序保持）
并发支持 不安全 同样不安全，但优化冲突性能

✅ 15. JDK1.7 vs 1.8 虚拟机变化（JVM）

变化点 JDK7 JDK8
永久代 有 PermGen（类信息存放） ❌ 被移除
元空间 ❌ 无 ✅ Metaspace（使用 native 内存）
Lambda 表达式 ❌ 不支持 ✅ 支持（字节码层面调用 invokedynamic）

✅ 16. ArrayList vs LinkedList

特性 ArrayList LinkedList
底层结构 数组 双向链表
随机访问 ✅ 快 ❌ 慢
插入删除 ❌ 慢（涉及数组移动） ✅ 快（指针操作）
空间利用 连续空间 离散节点，略低

✅ 17. HashMap 的 put 方法简述（阿里一面常考）

  • 根据 key 的 hash 值计算桶索引

  • 若桶为空，直接插入；否则遍历链表/树

  • 若 key 已存在，覆盖 value

  • 若链表长度 > 8 且容量 ≥ 64，转为红黑树

  • 若 size 超过阈值，触发扩容

✅ 18. 接口 vs 抽象类

特性 接口（interface） 抽象类（abstract class）
多继承 ✅ 支持 ❌ 不支持
成员类型 常量 + 抽象方法（JDK8 支持 default） 可以有变量、构造器
使用场景 标准规范 公共实现逻辑复用

✅ 19. 泛型中的 extends vs super

关键字 表示 作用
<? extends T> 上界通配符 读取时安全（可向下转型），不能添加
<? super T> 下界通配符 添加元素安全（向上转型），读取需 Object 转换

✅ 20. 深拷贝 vs 浅拷贝

类型 含义 示例
浅拷贝 只复制对象的引用 clone() 默认行为
深拷贝 对象 + 嵌套引用都复制一份新内存 序列化 + 反序列化 或手动递归拷贝
