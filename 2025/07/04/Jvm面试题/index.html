Jvm面试题

July 04, 2025 2315

1. 类的生命周期

类从被加载到卸载经历加载、验证、准备、解析、初始化、使用、卸载阶段，卸载一般发生在类加载器被回收时（多见于动态加载场景）。

2. JVM 类加载机制

采用双亲委派模型，类加载请求先委托父加载器，保证核心类唯一和安全。类加载器分为引导、扩展、应用和自定义加载器。

3. JVM 内存区域

包括堆（存对象实例，线程共享）、方法区（存类信息、常量池、静态变量）、Java虚拟机栈（方法调用帧，线程私有）、程序计数器（当前执行字节码地址）、本地方法栈。

4. 对象创建过程

类加载检查→堆内存分配（一般在 Eden 区，用 TLAB 优化）→初始化对象头→默认成员变量赋值→执行构造器→返回引用。

5. 对象内存分配方式

新生代 Eden 优先分配，大对象直接进入老年代；逃逸分析支持栈上分配；TLAB 减少多线程竞争。

6. JVM 创建对象如何解决多线程内存抢占？

通过线程私有的 TLAB 优化分配，避免竞争；不足时使用 CAS 保证堆上分配安全。

7. 对象内存布局和大小计算

对象头（Mark Word、类型指针） + 实例数据 + 对齐填充，大小受字段类型、顺序和压缩指针影响。

8. JVM 内存泄露原因

静态集合未清理、ThreadLocal 未及时删除、监听器未注销、缓存无过期策略、长生命周期对象引用短生命周期对象。

9. 如何判断对象仍然存活？

采用可达性分析，从 GC Roots 出发，能访问到的对象为存活。

10. 垃圾收集算法

标记-清除、复制、标记-整理、分代收集，各有优缺点。

11. 三色标记算法

对象分白（未标记）、灰（已标记未扫描）、黑（已扫描）三色，支持并发标记。缺陷是并发期间可能漏标，需写屏障修正。

12. CMS 收集器垃圾收集过程

初始标记（STW）→ 并发标记 → 预清理 → 重新标记（STW）→ 并发清理 → 并发重置。

13. G1 垃圾收集器

基于 Region 分区，支持并发标记和混合回收，用户可设定最大停顿时间，自动整理内存。

14. 为什么引入 G1？

为解决 CMS 的碎片、多次 STW、Full GC 频繁问题，提升吞吐和停顿控制。

15. 项目用什么垃圾收集器，为什么？

一般选 G1，适合大堆、低停顿和高并发场景。

16. 对象一定分配在堆中吗？

不一定，逃逸分析可使对象栈上分配，大对象可能直接进入老年代。

17. JVM 监控和故障处理工具

jps, jstack, jmap, jstat, jcmd, Arthas, VisualVM, MAT, GCeasy 等。

18. JVM 常见参数配置

  • 堆大小：-Xms（初始堆大小）、-Xmx（最大堆大小）

  • 年轻代大小：-Xmn

  • GC 策略：-XX:+UseG1GC、-XX:+UseConcMarkSweepGC等

  • GC 日志打印：-XX:+PrintGCDetails、-Xlog:gc（JDK9+）

  • 堆转储：-XX:+HeapDumpOnOutOfMemoryError

19. 有做过 JVM 调优吗？

通常会调整堆大小、年轻代比例，优化 GC 策略参数，结合业务分析堆 Dump 和线程 Dump，减少 Full GC，优化对象创建和缓存管理。

20. 线上 CPU 占用过高怎么排查？

  • 使用 top -H 查看线程CPU使用情况

  • jstack 导出线程堆栈，定位热点代码、死循环、锁竞争、GC 停顿

  • 结合业务日志分析异常请求

21. 内存飙高问题怎么排查？

  • 使用 jmap -histo 分析对象分布

  • 触发 Heap Dump，使用 MAT 等工具定位内存泄漏点

  • 重点关注缓存、线程池、连接池等资源管理

22. 频繁 Minor GC 怎么办？

  • 增大 Eden 区，减少新生代频繁回收

  • 减少短生命周期对象创建

  • 优化 TLAB 配置，减少内存分配竞争

23. 频繁 Full GC 怎么办？

  • 扩大老年代空间

  • 优化代码减少内存泄漏

  • 调整 CMS 或 G1 参数，避免并发失败

  • 监控元空间，防止溢出

24. 定位及解决 OOM

  • 配置堆转储 -XX:+HeapDumpOnOutOfMemoryError

  • 使用 MAT 等工具分析 Dump 文件，找出泄漏对象

  • 优化代码逻辑或增加内存配置

25. OOM 一定会导致 JVM 退出吗？

不一定。堆内存溢出通常可捕获；元空间或线程栈溢出多导致 JVM 崩溃。

26. 内存泄漏和内存溢出区别

  • 泄漏：程序无意中持有无用对象引用，内存长期占用。

  • 溢出：申请内存超出 JVM 最大限制，分配失败。

27. JVM 对象分配如何保证线程安全？

通过线程本地的 TLAB 缓冲区和底层 CAS 操作保证线程安全。

28. 堆一定是线程共享的吗？

是的，所有线程共享堆。

29. Class 常量池和运行时常量池区别

  • Class 文件中的常量池是静态的字节码结构。

  • 运行时常量池是方法区中动态生成的，支持运行时添加。

30. 运行时常量池和字符串常量池区别

运行时常量池存储所有常量和符号引用，字符串常量池是其中特化存放字符串字面量的区域。

31. 什么情况下会导致 JVM 退出？

主线程结束无非守护线程，调用 System.exit，严重 OOM，未捕获异常，系统或容器关闭。

32. JVM 内存为什么要分代？

不同生命周期对象采用不同回收策略，提高回收效率和减少停顿。

33. GC 是随时都能进行的吗？

不是，GC 触发基于内存状态和 JVM 策略，部分阶段需要暂停所有线程（STW）。
