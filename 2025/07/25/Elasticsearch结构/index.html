Elasticsearch 脑图结构

Elasticsearch
├── 一、核心概念
│   ├── Index（索引）
│   ├── Document（文档）
│   ├── Type（类型，7.x后废弃）
│   ├── Field（字段）
│   ├── Mapping（字段映射）
│   ├── Analyzer（分词器）
│   └── Query DSL（查询语法）

├── 二、架构原理
│   ├── 节点类型
│   │   ├── Master Node
│   │   ├── Data Node
│   │   ├── Ingest Node
│   │   └── Coordinating Node
│   ├── 分片与副本
│   │   ├── Primary Shard
│   │   └── Replica Shard
│   ├── 倒排索引
│   └── 写入流程（Indexing Flow）
│       ├── 写入 -> LogBuffer -> Lucene Segment
│       └── Refresh / Flush / Merge

├── 三、数据操作
│   ├── 创建/删除索引
│   ├── 增删改查文档（CRUD）
│   ├── 批量操作（Bulk API）
│   ├── 条件查询（Match, Term, Range）
│   ├── 布尔组合（Bool Query）
│   └── 分页、排序、高亮

├── 四、分词器与搜索原理
│   ├── IK、Standard、Pinyin、EdgeNGram
│   ├── Tokenizer / Filter / CharFilter
│   └── 自定义分词器

├── 五、聚合分析（Aggregation）
│   ├── Bucket 聚合（Terms, Range, DateHistogram）
│   ├── Metric 聚合（Avg, Sum, Max, Min）
│   ├── Pipeline 聚合
│   └── 子聚合

├── 六、性能优化
│   ├── 查询性能
│   │   ├── 精准字段类型（keyword vs text）
│   │   ├── 使用 filter 缓存
│   │   ├── 限制返回字段和分页大小
│   ├── 写入优化
│   │   ├── Bulk 批量写
│   │   ├── 控制刷新策略（refresh_interval）
│   └── 索引优化
│       ├── Mapping 精简
│       └── Segment 合并

├── 七、集成与生态
│   ├── Logstash（日志收集）
│   ├── Beats（轻量收集器）
│   ├── Kibana（可视化）
│   ├── Elasticsearch Java REST Client
│   ├── Spring Data Elasticsearch
│   └── Fluent DSL（如：ElasticsearchTemplate）

├── 八、版本变化（重要）
│   ├── 6.x: 单类型限制
│   ├── 7.x: 删除 type
│   ├── 8.x: 默认开启安全认证、加密通信
│   └── Lucene 升级同步

├── 九、安全与权限
│   ├── 用户认证（Basic Auth、LDAP）
│   ├── 角色管理
│   ├── Field / Index Level 安全控制
│   └── TLS 通信加密

├── 十、典型应用场景
│   ├── 日志检索（ELK Stack）
│   ├── 全文搜索（商品/文档搜索）
│   ├── 实时监控告警
│   ├── 数据聚合分析
│   └── 推荐与相关性打分
