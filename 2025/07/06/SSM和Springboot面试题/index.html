Part 1：Spring 核心概念与设计模式

🔹 什么是 Spring？

Spring 是一个开源的轻量级 Java 企业级开发框架，核心目标是实现 IoC（控制反转） 和 AOP（面向切面编程），帮助开发者降低耦合、提升开发效率，构建松耦合、高内聚的企业应用。

🔹 什么是 Bean 的自动装配？有哪些方式？

Bean 自动装配是指 Spring 自动将满足条件的 Bean 注入到依赖的地方。

常见方式有：

  • @Autowired：按类型注入

  • @Qualifier：配合 @Autowired 按名称注入

  • @Resource：按名称注入（来自 JSR-250）

  • 构造器注入（推荐，适用于不可变依赖）

🔹 如何理解 Spring Boot 的 Starter？

starter 是 Spring Boot 的模块化依赖规范，每个 Starter 提供一组自动配置好的依赖和默认配置，让你“开箱即用”。

例子：

封装了：

  • Spring MVC

  • Jackson

  • 嵌入式 Tomcat 等

🔹 如何实现一个 IOC 容器？

简单版实现：

  1. 解析配置文件或注解

  2. 实例化 Bean（反射）

  3. 完成依赖注入（setter / 构造器）

  4. 存入 Bean 容器（Map）

  5. 支持 Aware 回调、生命周期、单例池管理等

🔹 对 IOC 的理解？

IOC（Inversion of Control）即控制反转，是一种由容器负责对象的创建与依赖管理的思想。开发者无需自己 new 对象，Spring 容器负责控制对象的生命周期与依赖注入。

🔹 Spring 用到了哪些设计模式？

  • 工厂模式：创建 Bean 对象

  • 单例模式：默认 Bean 是单例

  • 代理模式：AOP、事务增强

  • 模板方法模式：JdbcTemplate 等封装流程

  • 策略模式：Resource、事务管理器选择

  • 观察者模式：事件发布机制（ApplicationEvent）

🔹 简述 Spring Bean 的生命周期

  1. 实例化（反射）

  2. 依赖注入

  3. Aware 接口回调（如 BeanNameAware）

  4. BeanPostProcessor 前置处理

  5. 初始化（@PostConstruct、InitializingBean）

  6. BeanPostProcessor 后置处理

  7. Bean 可用

  8. 销毁（@PreDestroy、DisposableBean）

🔹 Spring 如何处理循环依赖？

对于单例 Bean，Spring 采用三级缓存解决循环依赖：

  1. 一级缓存：成品对象

  2. 二级缓存：提前暴露的半成品（早期对象）

  3. 三级缓存：ObjectFactory（创建 Bean 的回调）

通过提前暴露引用，避免构造死循环。⚠️ 不支持构造器循环依赖。

🔹 Spring 中后置处理器的作用？

BeanPostProcessor 是 Spring 提供的扩展接口，可在 Bean 初始化前后做增强：

  • 创建代理对象（AOP）

  • 注解处理（如 @Autowired）

  • 自定义增强逻辑

Part 2：AOP 与事务机制

🔹 对 AOP 的理解

AOP（面向切面编程）用于横切关注点的解耦，如日志、权限、事务。通过代理对象在方法执行前后插入逻辑。

核心术语：

  • JoinPoint：连接点

  • Pointcut：切入点表达式

  • Advice：增强逻辑

  • Aspect：切面类

  • Weaving：织入代理过程

🔹 如何实现 AOP，项目哪些地方用到了 AOP？

  • 基于 JDK 动态代理（接口）或 CGLIB 代理（类）

  • 实现方式：@Aspect + @Around / @Before / @After

  • 项目中常见应用：

    • 日志收集

    • 数据权限校验

    • 异常报警

    • Redis 缓存切面

    • @Transactional

🔹 Spring 的事务机制？

基于 AOP 实现，使用 @Transactional 实现声明式事务控制，事务拦截器负责在方法调用前后开启、提交或回滚事务。

🔹 Spring 中 @Transactional 失效的几种场景？

  1. 自调用失效：本类方法调用不会触发代理；

  2. 方法不是 public；

  3. 异常未抛出或为非运行时异常；

  4. 配置错误：如未扫描 @EnableTransactionManagement；

  5. 代理对象未生效：未通过 Spring 容器创建 Bean。

🔹 Spring 的事务传播机制（常用）？

  • REQUIRED（默认）：有就加入，无则新建

  • REQUIRES_NEW：每次都新建

  • NESTED：嵌套事务（依赖保存点）

  • SUPPORTS：有就用，没有就不用

  • NOT_SUPPORTED：挂起事务

  • NEVER：必须无事务

🔹 Spring 中事务实现方式和隔离级别？

  • 事务通过 TransactionInterceptor 统一处理；

  • 底层调用数据库连接的事务控制；

  • 隔离级别包括：

    • READ_COMMITTED

    • REPEATABLE_READ

    • SERIALIZABLE

    • READ_UNCOMMITTED（不推荐）

Part 3：Spring Boot 原理与注解

🔹 Spring Boot 自动配置原理？

  • @SpringBootApplication = @EnableAutoConfiguration

  • 底层使用 spring.factories SPI 机制加载 @Configuration 配置类

  • 条件装配：@ConditionalOnXxx

🔹 常见 Spring Boot 注解及实现？

注解 作用 底层实现
@SpringBootApplication 启动入口，自动装配 @ComponentScan + @EnableAutoConfiguration
@ConfigurationProperties 属性绑定 反射绑定字段
@EnableAutoConfiguration 加载自动配置 spring.factories
@ConditionalOnXxx 条件加载 Bean Condition 接口判断

🔹 Spring Boot 是如何启动 Tomcat 的？

  • 引入 spring-boot-starter-web；

  • 自动装配 EmbeddedWebServerFactory（Tomcat 工厂）；

  • 调用 TomcatWebServer.start() 启动内嵌服务器；

  • 自动扫描 @RestController / @RequestMapping 处理请求。

🔹 Spring Boot 配置文件加载顺序？

优先级从高到低（高的覆盖低的）：

  1. 命令行参数

  2. application.properties / yaml（resources/config/ > resources/）

  3. 外部配置（环境变量、系统变量）

  4. @PropertySource

  5. 默认配置文件

Part 4：Spring MVC 流程与组件

🔹 SpringMVC 工作流程？

  1. 请求由 DispatcherServlet 接收；

  2. 通过 HandlerMapping 找到目标 Controller；

  3. 执行目标方法（HandlerAdapter）；

  4. 返回 ModelAndView；

  5. 渲染 View；

  6. 响应结果返回客户端。

🔹 SpringMVC 九大组件？

  1. DispatcherServlet

  2. HandlerMapping

  3. HandlerAdapter

  4. Controller

  5. ViewResolver

  6. View

  7. ModelAndView

  8. LocaleResolver

  9. ExceptionResolver

🔹 控制器是单例的吗？

是的，SpringMVC 默认 Controller 是单例，但线程不安全，Spring 通过将请求参数放在 ThreadLocal 中解决线程隔离问题。

🔹 Spring、SpringMVC、SpringBoot 区别？

特性 Spring SpringMVC Spring Boot
定位 容器 + AOP + DI Web 请求处理框架 全家桶快速开发框架
关注点 Bean 生命周期、AOP 路由、Controller 调度 自动配置、内嵌容器
配置复杂度 高 中 低

Part 5：MyBatis 核心原理与用法

🔹 #{} 和 ${} 的区别？

  • #{}： 预编译参数，防 SQL 注入（推荐）

  • ${}：字符串拼接，适用于动态表名、列名

🔹 MyBatis 插件运行原理及开发流程？

  • 拦截器基于责任链模式；

  • 拦截点：Executor、StatementHandler、ParameterHandler、ResultSetHandler；

  • 插件注册：@Intercepts({@Signature(...)})

🔹 MyBatis 优点与缺点？

优点：

  • SQL 灵活可控

  • 学习曲线平缓

  • 易于扩展（插件）

缺点：

  • 手写 SQL 成本高

  • 维护复杂度上升

  • 动态 SQL 可读性差

🔹 MyBatis 与 Hibernate 的区别？

项目 MyBatis Hibernate
类型 半自动 ORM 全自动 ORM
SQL 控制 手写 SQL 自动生成
性能 SQL 优化强 性能依赖缓存和映射
动态支持 强 弱

🔹 ApplicationContext 和 BeanFactory 区别？

  • BeanFactory 是最基本的 IoC 容器（懒加载）；

  • ApplicationContext 是增强型容器，支持：

    • 国际化

    • 事件发布

    • AOP

    • Bean 后处理器
