🔧 五、注意事项

  1. **ZSet 是基于跳表实现的，插入/查询性能非常高，O(log N)**。

  2. 默认排序是按 score 升序，可用 ZREVRANGE 变为降序。

  3. 元素数量大时，可以结合定期 ZREMRANGEBYRANK 清理尾部数据。

  4. Redis 是内存型数据库，排行榜需控制大小，避免内存膨胀。
