Mysql8.0新增特性详解

May 25, 2024 2308

新增特性

🚀 一、SQL & 表达能力增强

1. 窗口函数（Window Functions）

支持 OVER 子句：如 ROW_NUMBER()、RANK()、DENSE_RANK()、LEAD()、LAG() 等。

📌 用途：分组排名、同组统计，不再需要复杂子查询。

2. 公共表表达式（CTE / WITH 子句）

支持递归和非递归 CTE，增强了 SQL 的模块化与可读性。

📌 用途：组织层级、树形结构遍历。

3. 默认表达式和 INVISIBLE 列

  • 支持列默认值为表达式，如 DEFAULT (UUID())

  • 隐藏列（INVISIBLE）：对应用透明，不暴露于 SELECT *。

4. CHECK 约束

📌 之前版本忽略 CHECK 子句，现在强制生效。

⚡ 二、性能与可扩展性提升

1. 更强的 InnoDB 引擎

  • 原生支持 Atomic DDL（原子表结构变更）

  • Undo Tablespace 独立管理、自动清理

  • 更高效的 自适应哈希索引 和 buffer pool 并发度

  • 表空间 ENCRYPTION 支持动态加密

2. 历史数据闪回（UNDO 历史快照）

结合 TRANSACTION READ ONLY WITH CONSISTENT SNAPSHOT 实现数据时点读取，支持增强的数据恢复能力。

3. 更快的 DDL 操作

  • 支持 INSTANT ADD COLUMN，秒级添加列。

  • ALTER TABLE ... DROP COLUMN 性能更优。

  • 不再大范围复制数据页，效率大提升。

三、JSON 支持增强

MySQL 5.7 引入 JSON 类型，8.0 将其升级为“一等公民”：

新函数 作用
->> 提取并自动转为文本
JSON_TABLE() 把 JSON 转换为关系型表，极强！
JSON_ARRAYAGG() / JSON_OBJECTAGG() 类似 GROUP_CONCAT，用于 JSON 聚合
IS JSON 判断字段是否为合法 JSON
JSON_SCHEMA_VALID() 校验 JSON 是否符合 schema

📌 适用场景：NoSQL 混合、配置表、消息结构存储。

🔐 四、安全增强

1. 默认使用 caching_sha2_password 插件

更安全、更现代的密码加密方式，取代 mysql_native_password。

2. 支持角色（ROLES）管理

📌 权限集中管理更方便。

3. 支持密码历史、过期策略

📊 五、可观测性增强（DBA 福音）

1. Performance Schema / INFORMATION_SCHEMA 扩展

  • 新增多张表，统计更细化（如 io_latency、user_statistics）

  • 更丰富的 Wait/Lock 分析

2. 系统变量动态修改

很多变量可 SET GLOBAL 修改，无需重启。

3. 重做日志（redo log）支持压缩

降低 IO 压力，写入更快。

🌍 六、GIS（地理空间）支持增强

  • 支持更精确的空间计算（ST_ 前缀函数）

  • 支持 R-Tree 索引

  • 更好兼容 OpenGIS 标准

⚙ 七、复制与高可用增强

1. GTID 支持更完善

跨版本/重启后自动维护一致性。

2. 组复制（Group Replication）

支持自动主选举，结合 InnoDB Cluster 实现自动 Failover。

🧪 八、开发者体验提升

  • 更强的 EXPLAIN ANALYZE：真实执行时间分析

  • DESC FORMAT=JSON 输出结构化执行计划

  • 新增 RESOURCE GROUP：可限制某用户 CPU 资源

🎯 总结建议

使用场景 推荐功能
多层级数据 CTE（递归）
分组统计、排名 窗口函数
JSON 数据存储 JSON_TABLE、IS JSON、聚合函数
安全管理 角色系统、密码策略
大表变更 INSTANT ALTER COLUMN、Atomic DDL
高可用架构 Group Replication、GTID
性能优化 Performance Schema、EXPLAIN ANALYZE
